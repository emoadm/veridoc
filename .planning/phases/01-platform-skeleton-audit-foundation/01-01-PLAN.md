Scaffold the greenfield VeriDoc monorepo from nothing: a per-language workspace
(uv for Python, pnpm for React/TS) glued by a Taskfile, with the test framework
(pytest + Vitest) and Wave 0 test infrastructure that every later plan in this

libs (D-07) and the one reference service so downstream plans drop code into

test harness exist. This is the foundation all of Phase 1 stands on (PLAT-01,

lands in plan 06).

| Boundary | Description |
|----------|-------------|
| developer machine → public package registries (PyPI/npm) | Untrusted dependency artifacts cross into the build |
| CI runner → repo source | Build executes committed config/scripts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-SC | Tampering | uv add / pnpm add of [ASSUMED] packages | mitigate | Task 1 blocking-human package-legitimacy review; only APPROVED packages installed; rfc8785 individually adjudicated; lockfiles (uv.lock, pnpm-lock.yaml) committed |
| T-01-01 | Tampering | dependency version drift | mitigate | Pin versions in pyproject.toml/package.json + commit lockfiles; .tool-versions pins toolchain |
| T-01-02 | Information Disclosure | secrets committed to git | mitigate | .gitignore excludes env/secret files; no secrets introduced in this scaffold plan (secret contract handled in plan 06) |
| T-01-SC2 | Tampering | malicious postinstall scripts | accept | Single-cluster build milestone, no production deploy; lockfile pinning limits blast radius; revisit with pnpm `--ignore-scripts` policy if needed |
</threat_model>
