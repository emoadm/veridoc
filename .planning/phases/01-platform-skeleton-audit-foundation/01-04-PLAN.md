<objective>
Build veridoc-auth + veridoc-tenancy shared middleware (PLAT-03, D-01/D-02/D-03) and
commit the Keycloak realm-as-code. veridoc-auth verifies Keycloak-issued JWTs against
JWKS (signature + iss/aud/exp), enforces MFA via the acr/amr claim, and provides
deny-by-default 8-role RBAC plus a data-driven IP-allowlist hook. veridoc-tenancy
carries request-scoped site/study context (from Keycloak claims) into a fail-closed
contextvar that the data layer reads to scope every query. The Keycloak realm export
(8 roles, MFA flow, clients) is committed so CI/fresh clusters reproduce identity
deterministically.

Purpose: Identity and access are the Part 11 / GDPR / HIPAA core. The API tier NEVER
handles passwords — Keycloak owns credentials/MFA/sessions (D-01); the service only
validates JWTs. Realm-as-code prevents the "works on my cluster" auth gap (Pitfall 4).
Built once here, every later service imports these libs.
Output: Tested veridoc-auth + veridoc-tenancy libs, a committed realm JSON, and an
8-role RBAC matrix. (Live Keycloak integration — token round-trip in a real realm —
is exercised end-to-end in plan 05's reference service.)
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API (JWT) | Untrusted bearer tokens cross here; must be signature-verified |
| Keycloak → API | JWKS is the trust anchor for token signatures |
| tenant A request → tenant B data | Cross-tenant access must be impossible |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing / Elevation | JWT forged / alg=none / unsigned accepted | mitigate | Verify signature vs JWKS, pin RS256, check iss/aud/exp (middleware.py); test_jwt_verify covers each |
| T-04-02 | Spoofing | MFA bypass (token without MFA) | mitigate | Enforce acr/amr MFA claim in API middleware, not only Keycloak policy; test asserts missing-MFA rejected |
| T-04-03 | Elevation of Privilege | role escalation / missing access check | mitigate | Deny-by-default require_role across 8 roles (rbac.py); test_rbac asserts cross-role 403 |
| T-04-04 | Information Disclosure / Elevation | cross-tenant data access | mitigate | Fail-closed tenancy context (current_tenant raises if unset); contextvar isolation; data layer scopes by tenant |
| T-04-05 | Tampering | realm config drift / not reproducible (auth unconfigured in CI) | mitigate | Realm-as-code committed JSON imported on startup (Pitfall 4); change-controlled |
| T-04-06 | Information Disclosure | Keycloak admin / client secrets in git | mitigate | Realm JSON carries no plaintext secrets; client secrets injected via K8s Secrets in plan 06 |
| T-04-07 | Spoofing | IP-allowlist hook misconfigured/bypassed | accept | Hook wired data-driven this phase; full allowlist policy enforcement is a later hardening — documented |
</threat_model>
