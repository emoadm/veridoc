<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI runner → cluster | Untrusted build artifacts deployed; secrets injected here |
| git repo → deployed config | Committed Helm/IaC drives what runs; must carry no secrets |
| ingress → cluster | TLS termination boundary (design: TLS 1.3) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | secrets in Helm values / git | mitigate | secrets.yaml references K8s Secrets by name only; grep gate; SECRETS-CONTRACT.md; no plaintext secrets committed |
| T-06-02 | Tampering | deploy mocked/dry-run → unproven deploy path | mitigate | CI does a real helm install into a real kind cluster + kubectl wait (Anti-Pattern avoided, Success Criterion #1) |
| T-06-03 | Tampering / Repudiation | tamper-detection not actually gating | mitigate | CI runs test_mutated_row_breaks_chain against the deployed stack as a required job (phase gate) |
| T-06-04 | Tampering | realm not imported → auth unconfigured but "passes" | mitigate | Keycloak template imports veridoc-realm.json; integration tests exercise real tokens (Pitfall 4) |
| T-06-05 | Spoofing | cloud lock-in via provider-specific IaC | mitigate | Provider-portable Helm + thin provider-agnostic Terraform; no AWS/Azure resources (DEC-cloud-provider open) |
| T-06-06 | Tampering | unpinned CI actions / supply-chain | mitigate | Pin GitHub Action versions; reuse the package-legitimacy posture from plan 01 (T-01-SC) |
| T-06-07 | Denial of Service | data-in-transit interception | accept | TLS 1.3 at ingress is design-noted; full ingress/cert wiring is a later hardening this milestone documents |
</threat_model>
