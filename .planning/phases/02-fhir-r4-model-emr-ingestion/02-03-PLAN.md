spec-native provenance (D-03): R4B-only re-exports of the 9 resource types, an

legibility flags. This is the contract the agent fleet (Phases 4-6) reads from.

It is the dependency of every adapter (02-05) and the service (02-06).
Output: veridoc-fhir lib with models/repository/provenance/extensions, fully tested
(R4B construction, Mongo round-trip via testcontainer, provenance factory).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| adapter output → MongoDB | FHIR JSON documents persisted to the clinical store |
| FHIR JSON → fhir.resources validation | untrusted resource shape validated before storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-FHIR-01 | Tampering | unvalidated FHIR JSON stored to Mongo | mitigate | save() takes a fhir.resources.R4B model instance (Pydantic v2 full-spec validated), never a raw dict (ASVS V5) |
| T-02-FHIR-02 | Tampering | accidental R5 model use (Pitfall 1) | mitigate | models.py imports only fhir.resources.R4B.*; __all__ gate + source-inspection test asserts R4B |
| T-02-FHIR-03 | Information Disclosure | PII in stored FHIR documents | mitigate | provenance.entity.what.reference uses the pseudonymized reference, never natural_id; PII pseudonymization enforced upstream in adapters (02-05) |
| T-02-FHIR-04 | Denial of Service | unindexed COLLSCAN queries | mitigate | create_indexes() compound-indexes (resourceType,id) + (resourceType,subject.reference) at startup (Pitfall 6) |
</threat_model>
