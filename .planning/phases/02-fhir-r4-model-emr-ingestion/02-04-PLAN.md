<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted document bytes → OcrEngine | scanned-image bytes read by PIL/pytesseract |
| ingestion → BlobStore | original documents stored to object storage |
| OCR output → ALCOA flags | confidence-derived flags must be server-computed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-OCR-01 | Tampering | OCR confidence bypass | mitigate | flagged/escalated computed server-side from raw pytesseract output in OcrResult; never user-supplied |
| T-02-OCR-02 | Tampering | malicious document upload | mitigate | document bytes only read as image data by PIL/pytesseract, never executed; stored to blob, not run |
| T-02-BLOB-01 | Information Disclosure | blob object enumeration | mitigate | keys are non-guessable (UUID + site_id); bucket has no public access; boto3 authenticated client only |
| T-02-BLOB-02 | Information Disclosure | hardcoded cloud binding | mitigate | S3BlobStore endpoint_url conditional keeps DEC-cloud-provider OPEN; Azure path stubbed not bound |
</threat_model>
