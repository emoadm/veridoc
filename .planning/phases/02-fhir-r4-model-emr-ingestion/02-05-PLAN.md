fhir.resources.R4B models with pseudonymized PII, fully tested per the 02-VALIDATION map.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external HL7 message → hl7apy parser | untrusted ER7-encoded text |
| external PDF/Excel → pypdf/openpyxl | untrusted document parsing |
| external scanned image → OCR adapter | untrusted image bytes |
| raw PII → pseudonymization | PHI must not reach Mongo/blob in clear |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-ADP-01 | Information Disclosure | PII in adapter output (GDPR Art. 9) | mitigate | every adapter calls pseudonym_token and replaces name/dob/identifiers before returning resources; test_pseudonymization asserts raw MRN absent (SC-4) |
| T-02-ADP-02 | Tampering | malformed HL7 / XXE / parser abuse | mitigate | hl7apy structured parsing (no manual ER7); mapping only reads known segments; reject unknown message types with a clear error |
| T-02-ADP-03 | Tampering | zip-bomb / malformed PDF/Excel | mitigate | pypdf/openpyxl read-only parsing; size/page bounds enforced before extraction; failures surface as handled errors not crashes |
| T-02-ADP-04 | Tampering | OCR confidence forged by caller | mitigate | confidence + legibility flags derived server-side from OcrEngine output, never from request input (SC-3b) |
| T-02-ADP-05 | Information Disclosure | original document retains PII in blob | accept | original is retained for ALCOA+ Original principle; blob keys non-guessable + bucket non-public; access authenticated (mitigation in 02-04) |
</threat_model>
