<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /ingest | untrusted document upload crosses into the service |
| Redis → RQ worker | job payloads deserialized in a forked worker |
| worker → Postgres audit chain | worker-owned transaction writes the chain head |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-SVC-01 | Spoofing/Elevation | /ingest auth + RBAC | mitigate | Depends(principal_dependency) RS256/MFA + require_write_role deny-by-default (site-coordinator/data-manager) reused from Phase 1 (ASVS V2/V4) |
| T-02-SVC-02 | Information Disclosure | cross-tenant ingest | mitigate | current_tenant() fail-closed → 401 if unresolved; every job carries tenant_id (Phase 1 tenancy isolation) |
| T-02-SVC-03 | Tampering/Elevation | pickle RCE via RQ | mitigate | Queue + Worker both use rq.serializers.JSONSerializer; job args are primitives only (Pitfall 3) |
| T-02-SVC-04 | Repudiation | missing audit on ingest | mitigate | same-txn "ingest:enqueued" at the API + worker-owned "ingest:completed" both via append_audit (D-06); 15-yr retention chain (PLAT-02) |
| T-02-SVC-05 | Tampering | TesseractNotFoundError at runtime | mitigate | Dockerfile installs tesseract-ocr in builder + runtime (Pitfall 8) |
| T-02-SC | Tampering | image build pulls deps | mitigate | image builds from the committed uv.lock (PACKAGE-LEGITIMACY-approved in 02-01) |
</threat_model>
