<objective>
Implement the MDRWS boundary inside `veridoc-rave`: `odm.py` (parse ODM-XML → typed DTOs,
serialize DTOs → ODM-XML WRITE payloads, pseudonymizing SubjectKey at the boundary) and
`MdrwsHttpAdapter` (the single concrete `RavePort` using httpx + HTTP Basic auth). ODM-XML
and HTTP details stay entirely inside this layer (D-05); the adapter returns only DTOs.

Purpose: This is the production HTTP code path. Per D-01/SC-4 the SAME adapter is exercised
against the mock (Plan 03 + CI) and against real Rave — the mock→prod swap is a base_url
change in RaveProfile. Parsing is namespace-aware (Clark notation) to avoid the silent-miss
pitfall; WRITE payloads carry explicit TransactionType on every level.

Output: `odm.py`, `adapters/mdrws_http.py`, adapter export wired into the lib `__init__.py`,
and `test_odm.py` proving all 6 READ DTO parses + pseudonymization + malformed-input handling.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| adapter → MDRWS (mock/prod) | outbound HTTP carrying Basic credentials; untrusted ODM-XML response returns |
| adapter → agent/service code | only DTOs cross; ODM-XML + natural SubjectKey + credentials must not |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03 | Information Disclosure | Rave Subject ID leaking to DTOs/logs | mitigate | odm.py pseudonymizes SubjectKey via patient_pseudonym at the parse boundary (D-07/Pitfall 7) and discards the natural key; a test asserts the natural key is absent from returned DTOs. |
| T-03-04 | Tampering | XML injection in WRITE ODM payloads | mitigate | build_* helpers use the lxml etree builder, not f-string concatenation (Don't-Hand-Roll / Pitfall on string templates); user-supplied query text is set as element attributes/values via the builder, not interpolated into markup. |
| T-03-05 | Information Disclosure | Basic-auth credentials in logs | mitigate | Credentials live only on the RaveProfile and the per-request Authorization header; not logged. Mock/CI runs over cluster-internal HTTP; production MDRWS is HTTPS (TLS in transit). |
| T-03-06 | Tampering | Silent ODM mis-parse (namespace miss) | mitigate | Clark-notation parsing only (Pitfall 1); extra="forbid" DTOs (Plan 01) fail fast on unexpected shapes; malformed XML raises a typed error. |
</threat_model>
