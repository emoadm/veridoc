<objective>
Build the stateful Rave mock as a separate FastAPI service (D-02/D-03/D-04) emulating the
scoped MDRWS surface — real RWS URL conventions, HTTP Basic auth, ODM-XML responses, and
ODM-based error semantics — and wire the lib integration tests that exercise the REAL
`MdrwsHttpAdapter` against the running mock (SC-1 READ, SC-2 WRITE→read-back round-trip).

Purpose: The mock makes SC-1/SC-2 reproducibly verifiable without production Rave
(CON-medidata-partner). Per D-01 the adapter under test is the production HTTP adapter
pointed at the mock URL — no in-process fake — so the same code path ships to production.
Statefulness (D-04) is what makes WRITE verifiable: open a query, then read its status back.

Output: `services/rave-mock` service (state, odm_builder, MDRWS routers, main), workspace
registration, and the lib conftest + test_adapter_read + test_adapter_write integration tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → mock MDRWS endpoints | inbound HTTP carrying Basic credentials + ODM WRITE payloads (untrusted) |
| mock → rave-integration webhook | outbound signed HTTP (mock is the webhook sender, D-09) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Spoofing | Mock Basic-auth check | mitigate | Mock validates HTTP Basic with constant-time `compare_digest` (Pitfall 6) and returns an ODM error element on failure — faithfully replicating MDRWS auth so the adapter is correct at production swap. |
| T-03-08 | Tampering | ODM WRITE payload into mock | mitigate | Mock parses with lxml (structured, no eval) and rejects payloads missing explicit TransactionType (Pitfall 3) so the adapter's WRITE contract is validated against real Rave semantics. |
| T-03-09 | Tampering | Mock XML responses (injection) | mitigate | odm_builder uses the lxml etree builder, never f-string XML (Don't-Hand-Roll). |
| T-03-10 | Repudiation | Cross-test state bleed | mitigate | MockState is instance-level on app.state with a DELETE /mock/reset + autouse teardown (Pitfall 4) — no module globals. |
</threat_model>
