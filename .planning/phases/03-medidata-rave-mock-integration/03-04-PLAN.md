authenticates inbound Rave events via HMAC-SHA256 (fail-closed), writes a `veridoc-audit`

JSON-serializable payload discipline are the load-bearing invariants.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Rave (mock/prod) → /webhook | inbound unauthenticated-by-default HTTP; the primary attack surface of this phase |
| webhook handler → rave-events queue | job payload crosses a process boundary into the RQ worker |
| handler/worker → Postgres audit | every event must be recorded immutably (PLAT-02) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-webhook-auth | Spoofing | POST /webhook (forged/unsigned event) | mitigate | HMAC-SHA256 over the raw body with a per-environment shared secret, compared via `hmac.compare_digest`; 401 + no audit + no enqueue on failure (fail-closed). HIGH severity — blocks the plan if unmitigated. |
| T-03-11 | Elevation of Privilege | rave-events RQ payload (pickle RCE) | mitigate | Queue + Worker use JSONSerializer (DEC-rq-json-serializer); job args are only JSON-serializable primitives — no pickle, no bytes, no objects. |
| T-03-12 | Repudiation | Orphaned/stale audit vs enqueue | mitigate | append_audit → enqueue → session.commit() ordering (Pitfall 5 / DEC-audit-same-txn-writer); audit row + enqueue confirmation are atomic. |
| T-03-13 | Information Disclosure | Subject ID / secret in logs | mitigate | subject_key carried as the pseudonymized value from the event; rave_webhook_secret read from a K8s Secret at runtime, never committed (Plan 05 ephemeral generation). |
| T-03-14 | Tampering | Webhook timing/replay | accept | Replay/idempotency deferred to Phase 4 (RESEARCH.md Don't-Hand-Roll); Phase 3 stub does not require idempotency — documented, low residual risk in mock/CI. |
</threat_model>
